# ADR-034 — Client-Side Body Codec SPI + `KernelWebClient` Facade (link stub)

**Authoritative copy:** [`exeris-kernel/docs/adr/ADR-034-client-side-body-codec-spi.md`](../../../exeris-kernel/docs/adr/ADR-034-client-side-body-codec-spi.md)

ADR-034 is a **per-repo** decision owned by `exeris-kernel` (kernel/transport scope), with cross-repo coordination touching this repository for the `KernelClientGenerator` FQN swap. It supersedes ADR-026.

## Why this stub exists

`exeris-tooling/exeris-codegen-java/src/main/java/eu/exeris/tooling/codegen/java/kernel/KernelClientGenerator.java` references the kernel-side client facade via `ClassName.get(...)` constants (lines 43–46). ADR-034 changes those FQNs from `eu.exeris.kernel.transport.http3.client.ExerisWebClient` to `eu.exeris.kernel.core.http.client.KernelWebClient` (with the same nested `WebClientException`). This stub exists so generator authors and tooling reviewers find the kernel-side decision without searching the kernel repo.

> **Historical note:** an earlier hypothesis (the reverted PR #60 attempt) assumed the rename target was `eu.exeris.kernel.community.http.client.CommunityWebClient`. That class never shipped in any 0.8.0-SNAPSHOT jar — see the PR #60 thread for the diagnostic trail. ADR-034 landed the tier-neutral `KernelWebClient` facade in `core`, not in `community`.

## When to consult ADR-034 from tooling

- Any change to the generator's `WEB_CLIENT` / `WEB_CLIENT_EXCEPTION` constants — verify alignment with ADR-034 §1–4 (the tier-neutral facade lives in `eu.exeris.kernel.core.http.client`, not in Community).
- Any new generator that emits HTTP-client-shaped code (e.g., a TypeScript / Kotlin codegen track) — the same tier-neutral FQN discipline applies. The kernel-side facade name is `KernelWebClient` precisely so generator output never encodes Community / Enterprise tier identity in user-visible application source.
- Any proposal to emit code that bypasses the facade and calls `HttpClientEngine.send` directly — that codepath does not need the codec SPI but loses the typed binding ergonomics. ADR-034 §Alternatives A documents why the typed API does not live on the engine SPI itself.
- Any E2E generator fixture that compiles emitted client code against the kernel artifact — the fixture must target the new FQN (`KernelWebClient`) after the lockstep PR lands; pre-lockstep fixtures that import the legacy `eu.exeris.kernel.transport.http3.client.ExerisWebClient` will fail compilation against the new kernel artifact.

## Engineering status

ADR-034 is **ACCEPTED** (2026-05-19). The lockstep `KernelClientGenerator` FQN swap + class Javadoc / `@see` update + E2E fixture rename land as PR-C of the ADR-034 lockstep set described in ADR-034 §Implementation Plan. ADR-015 (Codegen Emission Strategy) governs the JavaPoet emission machinery itself — ADR-034 only updates the target FQN constants and the generator-class Javadoc that references the facade by name.
