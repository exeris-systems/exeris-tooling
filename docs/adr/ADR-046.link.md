# ADR-046 — Event-Payload Codec SPI (link stub)

**Authoritative copy:** [`exeris-kernel/docs/adr/ADR-046-event-payload-codec-spi.md`](https://github.com/exeris-systems/exeris-kernel/blob/main/docs/adr/ADR-046-event-payload-codec-spi.md)

ADR-046 is a **cross-repo** decision owned by `exeris-kernel`. It carries the body-codec matrix
(ADR-009/034/036) onto the **events** surface: a tier-neutral `EventPayloadCodec` +
`EventPayloadCodecRegistry` in `eu.exeris.kernel.spi.events.codec`, registry-selected by
`(payloadType, contentType)` with a Community JSON driver, resolved **in the generated
`*EventPublisher`** ("site B") so `EventBus`/`EventEngine` stay byte-for-byte unchanged. The wiring
rides a new optional `KernelProviders.EVENT_PAYLOAD_CODEC_REGISTRY` slot.

## Why this stub exists — `exeris-tooling` has a concrete obligation

ADR-046 unblocks the **EV1 runtime half**: before it, the generated publisher shipped
`EventPayload.empty()`, so domain events carried no data. The tooling deliverable (ADR-046 Engineering
Protocol, PR-B):

- **`KernelEventGenerator` rewrite** — for an event with `payloadFields`, the emitted
  `*EventPublisher.publish<Event>(UUID streamId, <Entity> entity)` builds a nested **redacted**
  `<Event>Payload` record (`payloadFields` minus `sensitiveFields`) from the aggregate's getters,
  resolves an `EventPayloadCodec` via `KernelProviders.eventPayloadCodecRegistry()` (default
  content-type `application/json`), and publishes the encoded `EventPayload`. Redaction is the
  publisher's job, applied **before** encode.
- **Fallback + JFR** — when the slot is unbound or no codec supports the payload, it falls back to
  `EventPayload.empty()`; the null-resolve branch emits a **producer-side** codec-resolution-failure
  JFR (the resolution happens in the producer, so this event ships here, not kernel-side).
- **The Wall** — the generated publisher names only SPI symbols
  (`KernelProviders` / `EventPayloadCodec` / `EventPayloadCodecRegistry` / `EventCodecContext` /
  `EventPayload`); never the `CommunityJsonEventPayloadCodec` driver or `tools.jackson.*`.

The `KernelCodegenCompileTest` fixture grew an `@DomainEvent` with `payloadFields` so the gate
`javac`-compiles the codec-resolved publisher against the live kernel codec SPI; `KernelEventGeneratorTest`
pins the redacted-record + resolution + Wall assertions.

The build-time half (processor resolving `payloadFields`/`sensitiveFields` into `DomainEventMetadata`,
the TS `event-gen` consuming them) ships in the same PR; the SDK `-io` reader mirror landed on the SDK
side (ADR-042).
